# Sub-Agent Doc: Image Processing Agent

본 문서는 **Image Processing Agent**가 담당해야 할 이미지 합성 및 오프스크린 오버레이 그래픽스 엔진의 구현 가이드를 정의합니다.

## 1. 역할 정의
- 촬영된 원본 고화질 이미지(Bitmap)를 가져와 필수 워터마크, 날짜/시간, 그리고 선택적 설정 데이터(메타데이터, 행정 주소)를 `react-native-view-shot` 기반 오버레이를 통해 합성하는 엔진 구현.
- 원본 이미지 가로세로 비율 대응 및 이미지 해상도 대비 텍스트/마크 요소 스케일링, 대비 조정 및 렌더링 품질 보장.

## 2. 합성 처리 파이프라인 (`ImageComposer`)

### 2.1 처리 구조 (React Native View/Image 기반 오버레이)
1. 합성용 오프스크린 컨테이너 컴포넌트(`ComposerOverlay`) 구현.
2. **배경 레이어**: 원본 이미지의 로컬 URI를 받아 `<Image>` 컴포넌트로 풀 스크린 배치.
3. **상단 오버레이 영역**:
   - **좌측 상단**: 고정 워터마크 `Timo` 텍스트 드로잉.
   - **우측 상단**: 촬영 시점 기준의 타임스탬프 (`yyyy-MM-dd HH:mm:ss`) 드로잉.
4. **하단 오버레이 영역**:
   - 설정 데이터 기반의 3줄 메타데이터 목록 조합 및 출력:
     - 1행: 기관명 / 담당자
     - 2행: 행정 주소 (위치 설정이 활성화되어 있고 주소 조회가 성공한 경우에만 노출)
     - 3행: 기타 메모
5. **디자인 테마 및 대비 처리**:
   - 메타데이터 배경 반투명 활성화(`metadataBackgroundEnabled`) 설정에 따라 어두운 반투명 블랙 백그라운드 배너 및 타임스탬프 배지 배경 색상 조절.
   - 가독성 확보를 위해 글자 뒤에 `textShadow` (그림자 효과) 적용.

### 2.2 고해상도 출력 및 스케일링
- 단순 모바일 화면 캡처 시 화질이 떨어지므로, `react-native-view-shot` 캡처 시 타깃 너비 `1920px` 수준의 초고해상도로 스케일링 비율을 연산하여 비트맵 화질을 확보합니다.
- `scale` 값을 소스 이미지 비율에 맞춰 폰트 사이즈 및 마진/패딩 크기에 곱해줌으로써 고품질 출력을 보장합니다.

## 3. 제약 및 주의사항
- **300줄 룰**: `ImageComposer` 및 컴포넌트 구조가 300줄을 넘지 않도록 가독성 있고 간결하게 작성합니다.
- **메모리 최적화**: 고해상도 이미지 다중 생성 중 OOM(Out of Memory) 방지를 위해 캡처 품질을 적절히 조율하고 불필요한 리렌더링을 차단합니다.

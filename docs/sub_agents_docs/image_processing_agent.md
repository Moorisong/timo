# Sub-Agent Doc: Image Processing Agent

본 문서는 **Image Processing Agent**가 담당해야 할 이미지 합성 및 Canvas 그래픽스 엔진의 구현 가이드를 정의합니다.

## 1. 역할 정의
- 촬영된 원본 고화질 이미지(Bitmap)를 가져와 필수 워터마크, 날짜/시간, 그리고 선택적 설정 데이터(메타데이터, 행정 주소)를 Canvas를 통해 합성하는 엔진 구현.
- 이미지 해상도 대비 텍스트 스케일링, 대비 조정 및 렌더링 품질 보장.

## 2. 합성 처리 파이프라인 (`ImageComposer`)

### 2.1 처리 순서
1. 원본 이미지 Bitmap 로드 및 Mutable Canvas 생성.
2. **우측 상단**: 워터마크 `Timo` 드로잉 (항상 포함, 고정 텍스트).
3. **워터마크 하단**: 촬영 시점 기준의 타임스탬프 (`yyyy-MM-dd HH:mm:ss`) 드로잉.
4. **하단 영역 (오버레이)**:
  - 설정에 따른 기관명(최대 20자), 담당자(최대 10자), 기타(최대 20자) 출력.
  - 위치 토글이 ON이고 주소가 수집된 경우, 행정 주소 텍스트 정보 출력.

### 2.2 텍스트 스타일 및 접근성
- 단일 Sans-serif 계열 폰트 사용.
- 이미지 밝기에 따른 가독성을 확보하기 위해 텍스트 뒤에 반투명 어두운 배경(배너/섀도우)을 적용하거나 자동 대비 처리를 수행합니다.
- 원본 이미지 해상도가 다를 경우에도 워터마크와 텍스트의 크기/위치가 일정 비율로 렌더링되도록 스케일 계산 공식을 적용해야 합니다.

## 3. 제약 및 주의사항
- **300줄 룰**: `ImageComposer` 내부 로직이 길어질 경우 `WatermarkRenderer`, `TimestampRenderer`, `MetadataRenderer` 등 기능별 렌더러로 서브 모듈을 분리합니다.
- **메모리 최적화**: 고해상도 Bitmap 합성 중 OutOfMemory(OOM)가 발생하지 않도록 메모리 해제 및 캔버스 비트맵 생성을 조심히 다루어야 합니다.

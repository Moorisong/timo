# 📱 APP_CONVENTIONS (앱 특화 규칙)

이 문서는 `React Native / Expo` 및 안드로이드/iOS 모바일 하이브리드 프로덕트 환경에만 특화되어 적용되는 개발 컨벤션이다.

---

## 📁 1. 크로스 플랫폼 및 UI 레이아웃
* **반응형 UI 구조 구현:** 모바일과 태블릿 등 다양한 스크린 해상도를 고려한 반응형 UI 레이아웃을 컴포넌트에 적용한다.
* **네이티브 브릿지 활용:** Expo 플러그인 및 네이티브 모듈 제어 시, 라이브러리 간의 의존성 충돌이 없도록 버전 사양을 철저히 검증하고 래퍼(Wrapper) 패턴을 통해 독립성을 유지한다.

---

## 💾 2. 온디바이스 데이터 관리 규칙 (묘록/티모 기반)
* **하루 1 Row 데이터 규칙 준수:** 장기 데이터 누적 시 성능 저하 및 차트 시각화 복잡도 증가를 방지하기 위해, 배변/구토 등 반복성 데이터는 누적 카운트 처리를 하고, 특정 수치는 덮어쓰기 방식으로 설계하여 하루에 단 하나의 행(1 Row)만 생성되도록 SQLite 및 원격 데이터베이스 구조를 강제한다.
* **안전한 다묘(Multi-Pet) 확장 구조:** 내부 모든 데이터 테이블 구조에는 펫 아이디(Pet ID) 필터를 인덱스로 포함시켜 필터링 효율을 높인다.
* **소프트 삭제(Soft Delete) 원칙:** 유저의 데이터는 중요한 자산이므로 정보 삭제 요청 시 영구 삭제하지 않고, `isDeleted`나 `deletedAt` 플래그를 활용한 소프트 삭제 방식을 적용해 데이터를 보존한다.

---

## 💻 3. 네트워크 독립형 및 비용 최적화 하드웨어 제어
* **온디바이스 독립형 아키텍처:** 현장직 유저의 불안정한 네트워크 환경에서도 앱이 완벽히 작동하도록 서버가 없는 온디바이스 독립형(Jetpack DataStore, SQLite, MediaStore 등 로컬 저장소 제어) 구조를 기반으로 코드를 작성한다.
* **하드웨어 레이어 결합 (CameraX & Canvas):** 가공되지 않은 촬영 이미지 위에 변조가 불가능한 고유 타임스탬프 텍스트를 드로잉하는 이미지 프로세싱 레이어 수정 시, 안드로이드 Canvas 및 CameraX 라이브러리의 생명주기와 리소스 해제를 철저히 관리하여 메모리 누수를 방지한다.
* **위치 데이터 정제 및 비용 절감:** API 호출 비용을 아끼기 위해 유저가 1미터 이하로 미세하게 움직였을 때는 이전 위치 주소를 재사용하고, 기지국 기반 위치를 빠르게 잡기 위해 2초 타임아웃 예외 처리를 반영한다. 가짜 GPS 조작 유저를 필터링하는 방어 코드를 반드시 유지한다.

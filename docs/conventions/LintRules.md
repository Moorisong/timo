# 🎨 LINT_RULES (초엄격 소스 코드 스타일 가이드)

본 문서는 프로젝트의 가독성, 안전성, 유지보수성을 극대화하기 위해 AI 에이전트가 100% 준수해야 하는 엄격한 서식 및 정적 분석 규칙이다.

---

## 🔤 1. 기본 서식 설정 (Basic Formatting)

* **들여쓰기(Indent):** 2 Spaces(공백 2칸)를 절대 원칙으로 하며, 탭(Tab) 사용을 금지한다.
* **세미콜론(Semicolon):** 모든 문장의 종결점에는 세미콜론(`;`)을 누락 없이 반드시 명시한다.
* **따옴표(Quotes):** 
  * TypeScript/JavaScript 일반 문자열은 Single Quotes(`'`)만 사용한다.
  * JSX/TSX 템플릿의 속성(Attribute) 내에서는 Double Quotes(`"`)를 허용한다.
  * 문자열 내부에 변수 삽입이 필요한 경우에만 템플릿 리터럴(Backticks: `` ` ``)을 사용한다.
* **최대 줄 길이(Line Length):** 코드 가독성을 위해 한 줄의 길이는 최대 120자를 넘지 않도록 적절히 줄바꿈한다.

---

## 🗂️ 2. 일관된 임포트(Import) 4단계 정렬 규칙

파일 최상단의 임포트 구문은 의존성 추적을 명확히 하기 위해 아래 그룹 순서대로 정렬하며, **그룹 간에는 반드시 공백(줄바꿈 1칸)**을 둔다. 각 그룹 내부에서는 알파벳 순서로 정렬한다.

1. **내장 및 외부 패키지 (Built-in / External):** React, Next.js, React Native 엔진 및 전역 라이브러리 (예: `react`, `next/link`, `zustand`, `expo`)
2. **절대 경로/별칭 모듈 (Absolute / Alias):** 프로젝트 루트 기준 별칭을 사용하는 내부 아키텍처 모듈 (예: `@/components`, `@/utils`, `@/hooks`, `@/stores`)
3. **상대 경로 모듈 (Relative):** 현재 파일과 인접한 위치의 로컬 모듈 및 컴포넌트 (예: `./Button`, `../types`)
4. **에셋 및 스타일 (Assets / Styles):** 이미지 파일, 전역/지역 스타일시트, 상수 정의 파일 (예: `@/assets/logo.png`, `./style.css`, `./constants`)

## 🛡️ 3. 초엄격 정적 타이핑 및 예외 규칙 (Anti-Pattern 차단)

* **`any` 타입 사용 절대 금지:** TypeScript 환경에서 임의의 편의를 위해 `any` 타입을 지정하는 것을 원천 금지한다. 타입을 확정할 수 없는 경우 언노운(`unknown`)을 사용하고 정밀하게 타입 가드(Type Guard)를 설계한다.
* **암묵적 타입 캐스팅 금지:** 눈에 보이지 않는 강제 형변환을 지양하고, 숫자 변환 시 `Number()` 등을 명시적으로 선언한다.
* **`console.log` 잔재 금지:** 디버깅을 위해 작성한 `console.log`, `console.dir` 등의 코드는 커밋 전에 무조건 모두 제거한다. 시스템 에러 로깅이 필요한 경우에만 `console.error` 또는 `warn`을 허용한다.
* **무의미한 변경 이력(Diff) 발생 방지:** 기존 소스 코드를 리팩토링할 때, 본인이 수정하지 않는 영역의 스타일 포맷을 임의로 대량 재정렬하여 Git Diff 이력을 지저분하게 만드는 행위를 금지한다.

## 🚀 4. 작업 완료 조건 및 린트 검증 프로세스

* 에이전트는 코드 작성을 마친 후, 사용자에게 보고하기 직전에 반드시 터미널에서 프로젝트의 정적 검증 스크립트(예: `npm run lint`, `yarn lint`)를 직접 호출해야 한다.
* 린트 검증 시 발생하는 **모든 에러(Error)와 경고(Warning)는 0개**여야 한다. "Warning은 괜찮겠지" 하고 방치한 상태로 보고하는 것을 절대 금지하며, 스스로 모든 경고를 해결한 뒤 깨끗한 상태에서만 최종 보고를 완료한다.

# AI Agent Working Rules

이 프로젝트에서 AI 에이전트는 아래 규칙을 반드시 따른다.

---

## 🚫 작업 권한 제한

- 커밋은 사용자가 명시적으로 요청할 때만 수행한다.
- 배포 또한 사용자가 명시적으로 요청할 때만 수행한다.

---

## 🧱 코드 구조 규칙

- 파일 하나당 300줄을 넘기지 않는다.
- 300줄을 초과할 가능성이 있으면 반드시 모듈 분리 및 구조 분해를 먼저 수행한다.
- 기능 단위로 파일을 분리하여 유지보수성을 높인다.

---

## 🧪 테스트 규칙

- 작업 완료를 사용자에게 보고하기 전에 반드시 테스트 코드를 실행한다.
- 모든 테스트가 통과한 경우에만 작업 완료를 보고한다.
- 테스트 실패 시, 보고하지 말고 먼저 수정한다.

---

## 🧹 코드 품질 규칙

- 사용하지 않는 변수는 반드시 제거한다.
- 반복적으로 사용되는 리터럴 값은 반드시 상수화한다.
- 반복되는 로직은 함수 또는 모듈로 분리하여 재사용한다.

---

## 💬 응답 언어 규칙

- 모든 응답은 한국어로 한다.
- 영어로 응답하지 않는다.

⚙️ 시스템 명령어 및 자동화 실행 규칙
이 프로젝트는 Antigravity 기반 자동화 환경을 사용한다.
실행 과정에서 시스템이 반복적으로 권한 승인(허용 여부)을 요청할 수 있다.
🚫 권한 요청 처리 규칙
일반적인 개발 작업(빌드, 파일 생성, 코드 수정, 리팩토링 등)에 대해서는 권한 요청을 하지 않는다.
사용자의 명시적 승인 없이도 안전한 범위 내 작업은 자동으로 실행한다.
동일하거나 반복적인 시스템 권한 요청 창이 발생하는 경우, 이를 중단 없이 자동으로 처리한다.
⚠️ 예외 사항
다음과 같은 경우에만 권한 요청을 허용한다:
시스템에 치명적인 영향을 줄 수 있는 삭제 작업
복구 불가능한 데이터 변경 작업
보안상 민감한 외부 접근 작업
🎯 목적
개발 흐름을 중단시키는 불필요한 권한 승인 과정을 제거하고,
에이전트가 작업을 연속적으로 수행할 수 있도록 한다.

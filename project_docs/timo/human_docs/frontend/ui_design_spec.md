# 🎨 페이지별 UI 구조 및 비즈니스 로직 명세 (`ui_design_spec.md`)

이 문서는 **Timo** 애플리케이션의 화면 구조(사이트맵), 각 페이지별 기능 명세, 비즈니스 로직 흐름 및 디자인 스타일 시스템 정보를 역추적하여 기술합니다.

---

## 🗺 서비스 사이트맵 및 라우팅 구조

Timo는 **Expo Router**를 이용한 파일 기반 라우팅을 채택하고 있으며, 라우터 경로는 `src/app/` 하위 파일들과 연동됩니다.

```text
/ (src/app/index.tsx) - 메인 카메라 화면 (CameraScreen)
  │
  ├───► /preview (src/app/preview.tsx) - 워터마크 합성 미리보기 및 작업 제어 (PreviewScreen)
  │
  └───► /settings (src/app/settings.tsx) - 워터마크 정보 및 오버레이 토글 설정 (SettingsScreen)
```

---

## 📱 페이지별 기능 및 비즈니스 로직 요약

### 1. 메인 카메라 화면 (`src/app/index.tsx`)
*   **핵심 기능:**
    *   실시간 디바이스 카메라 피드 제공 (`CameraView` 사용).
    *   상단 영역에 실시간 디지털 시계(초 단위 동적 갱신) 및 고유 워터마크(`Timo`) 표시.
    *   사용자가 지정한 위치 정보 활성화 옵션에 따라 하단에 행정 주소 및 GPS 상태를 실시간 수집 및 표기.
    *   플래시 및 셔터 애니메이션 깜빡임(Flash/Shutter animation)을 제거하여 현장 조사 작업 속도 최적화.
    *   개발환경(`__DEV__`) 한정 수동 GPS Mocking 동작 검증 버튼 제공.
*   **비즈니스 로직 흐름:**
    1.  화면 진입 시 `useFocusEffect`를 통해 `AsyncStorage`로부터 최신 설정 값 로드.
    2.  카메라 및 위치 수집(GPS) 권한 확인 및 요청.
    3.  촬영(Capture) 버튼 클릭 시, 카메라 프리뷰 스냅샷 획득 (`takePicture`).
    4.  획득된 원본 이미지 정보와 최신 설정, GPS 데이터를 `CaptureData` 객체로 패킹하여 JSON 문자열 형태로 미리보기 화면(`/preview`)에 전달.

### 2. 합성 미리보기 화면 (`src/app/preview.tsx`)
*   **핵심 기능:**
    *   화면에 표시된 레이아웃 해상도로 카메라 이미지에 날짜, 시간, 기관/담당자, 위치, 메모 등을 오버레이로 얹어 미리보기 제공.
    *   실제 이미지 저장 시에는 1920px 급의 초고해상도로 오프스크린 렌더러(`ViewShot`)를 구동하여 캡처를 수행함.
    *   성공적인 저장 시 토스트 알림을 띄우고 메인 카메라 화면으로 복귀.
    *   공유 기능을 통해 메신저, 이메일 등 타 앱으로 합성 완료된 이미지를 즉각 전송 가능.
*   **비즈니스 로직 흐름:**
    1.  이전 화면에서 전달받은 `CaptureData` JSON을 파싱하고, 이미지 해상도 비율(Aspect Ratio) 계산.
    2.  [저장] 버튼 선택 시 `isSavingRef`를 통한 중복 실행 방지 플래그 작동.
    3.  `react-native-view-shot`을 활용해 타겟 너비 1920px 기준의 합성 이미지를 캐시 파일로 렌더링.
    4.  `expo-media-library`를 호출하여 시스템 갤러리(DCIM)에 저장 후 메인 화면으로 전환.
    5.  [공유] 버튼 선택 시 `expo-sharing`을 사용하여 렌더링된 캐시 파일 주소 전달.

### 3. 설정 화면 (`src/app/settings.tsx`)
*   **핵심 기능:**
    *   촬영본 오버레이에 렌더링할 사용자 정보 입력란 제공 (기관명, 담당자명, 세부 현장 메모).
    *   한글/영어/이모지 혼용 시에도 바이트 오작동을 방지하는 유니코드 전용 글자수 카운팅 제공.
    *   오버레이 상세 디자인 토글 제어 (배경 블랙 박스 표시 유무, GPS 사용 유무).
    *   설정 변경 사항을 실시간 반영해 보여주는 동적 미니 프리뷰 박스 (`SettingsPreview`) 배치.
*   **비즈니스 로직 흐름:**
    1.  화면 렌더링 시 디바이스 스토리지에서 기존 키 값들을 불러와 폼에 채움.
    2.  TextInput의 `onChangeText` 실행 시 유니코드 검사를 통해 각 필드별 최대 길이(`agencyName`: 20자, `inspectorName`: 10자, `comment`: 20자) 내에서 잘라내기 적용.
    3.  [설정 저장] 버튼 터치 시 `AsyncStorage.multiSet`을 사용하여 디바이스 영속 디스크에 일괄 저장 후 이전 화면으로 이동.

## 🎨 하드코딩된 메인 컬러 및 디자인 테마

Timo의 UI 스타일링은 다크 스페이스 감성의 프리미엄 테마를 공통 채택하여 야외 현장 조사 상황에서 시인성을 높이고 불필요한 눈부심을 최소화합니다.

### 1. 전역 색상 상수 (`src/constants/colors.ts`)
*   **배경색 (Background):** `#000000` (순수 블랙)
*   **메인 포인트 컬러 (Primary):** `#3B82F6` (블루 계열) / 라이트 버전: `#60A5FA`
*   **기본 텍스트 (textPrimary):** `#FFFFFF` (순수 화이트)
*   **서브 텍스트 (textSecondary):** `rgba(255,255,255,0.6)` (반투명 화이트)
*   **카드/컴포넌트 표면 (Surface):** `rgba(255,255,255,0.12)` 및 `rgba(255,255,255,0.2)`
*   **오류 및 경고 (Error):** `#E57373` (연한 레드)
*   **경계선 (Border):** `rgba(255,255,255,0.15)`

### 2. 여백 및 패딩 구조 (`src/constants/theme.ts`)
*   **Spacing:** `half (2)`, `one (4)`, `two (8)`, `three (16)`, `four (24)`, `five (32)`, `six (64)`
*   **컨텐츠 최대 가로 너비 (MaxContentWidth):** `800` (태블릿 및 웹 화면 대응)

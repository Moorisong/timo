# Backend Local Data & Device Sensor Agent Reference

## 📝 1. 연동 기획 명세 (`backend_spec.md`, `database_schema.md`)
본 문서는 Timo 애플리케이션의 디바이스 시스템 브릿지 구성과 로컬 저장소 스키마 명세를 다룹니다.
- **아키텍처 구조:** 서버리스/클라이언트 단독(Serverless / Client-Only) 구조.
- **연동 센서 및 OS 모듈:** Reverse Geocoding(`expo-location`), 로컬 미디어 저장(`expo-media-library`).
- **영속 데이터 엔진:** `@react-native-async-storage/async-storage` 단일 레벨 키-밸류(Key-Value) 구조 사용.

## 🤖 2. AI 개발 지침 및 설계 구조
### 🎯 목적
서버 데이터베이스 통신 없이 디바이스 센서 데이터를 수집하고 기기 로컬 스토리지에 안정적으로 데이터를 영속화하는 데이터 파이프라인 및 백그라운드 모델 레이어를 설계합니다.

### 📦 패키지 및 타깃 경로 구조
- `src/services/`: 실제 센서 접근, 스토리지 입출력 및 미디어 저장 핸들러 모듈 배치
- `src/utils/`: 데이터 포매팅 및 유니코드 검증 함수 등 순수 함수 모음

### 🛠️ 개발 단계 (Step-by-Step 상세 로직)
1. **디바이스 OS 권한 연동:** 사용자 시스템 권한(Camera, Location, Media) 상태를 질의 및 요청하고, 거부 상태 처리 로직(예: 디바이스 시스템 설정 유도)을 `services` 레벨에서 구현합니다.
2. **Reverse Geocoding 인터페이스 구성:** 수집된 위도/경도를 `expo-location`을 이용해 OS에서 제공하는 한글 행정구역 주소 객체 형태로 안전하게 디코딩하고 예외 처리합니다.
3. **AsyncStorage 브릿지 및 스키마 검증:** `database_schema.md`에 명시된 키-밸류 구조를 바탕으로, 저장 전 `unicode.ts` 등을 거쳐 바이트 한계 유효성 검사를 수행한 뒤 AsyncStorage에 저장/조회하는 래퍼 함수를 구현합니다.
4. **미디어 I/O 연동:** `expo-media-library`와 `expo-sharing`을 사용하여 뷰에서 합성된 캐시 이미지를 로컬 갤러리로 안전하게 옮기거나 외부 앱으로 전송합니다.

## 🚨 3. 철벽 코드 컨벤션 및 제약 조건
- **[300줄 분리 규칙]:** 본 모듈의 실제 구현 소스 코드는 단일 파일 기준 **300줄**을 절대 초과할 수 없다. 구현 중 한계 도달 시 즉시 sub-module이나 utils로 함수를 분리 구조 분해할 것.
- **[로컬 종속성 유지]:** 원격 API, HTTP/Axios 통신 로직 추가 절대 금지. 모든 I/O 작업은 로컬 디바이스 범위에서 동기/비동기 처리로 완결할 것.
- **[보안 샌드박스 준수]:** 스토리지 저장 데이터는 민감 정보 관리에 유의하고, 안드로이드/iOS의 로컬 보안 정책 및 샌드박스 정책을 훼손하는 우회 로직을 구현하지 말 것.
- **[권한 거부 우회 제한]:** OS 시스템에서 반환된 권한 거부 상태를 강제로 무시하고 실행하는 코드를 작성해서는 안 되며, 반드시 우회 없는 명시적 분기 처리를 통해 안전하게 앱이 동작하도록 방어 코드를 적용할 것.
